Minhas "caixa de ferramentas"

Essa é a parte onde eu apresento minhas principais ferramentas de trabalho e como eu as uso para entregar aos clientes ótimos trabalhos. Fique à vontade para dar uma olhada, prometo que não vai ser chato.

HTML5

É a base de todo site que existe, todas as informações são exibidas através desta importante ferramenta.
Usando rotineiramente o HTML juntos com outras ferramentas eu faço excelentes conteúdos, tanto bonitos quanto funcionais.

CSS3

Estilo e beleza. É para isso que serve essa ferramenta.
Tudo que é bonito (ou nem tanto) que está nos sites é feito com CSS: cores, letras, imagens, enfim, tudo o que podemos ver nos sites é ajustado com CSS. Por isso sempre busco alcançar a maestria no uso dessa ferramenta para entregar ao cliente trabalhos bonitos e estilosos.

JavaScript

É a famosa linguagem da web, serve para dinamizar e automatizar praticamente qualquer coisa em um site.
Eu busco sempre novas tendências de uso dessa linguagem, inclusive por combinar com outras ferramentas

Node.js

É uma tecnologia em ascensão e serve para programar o que não vemos diretamente em um site.
Por exemplo: quando você aperta um botão com “todas as postagens” ou “salvar”, é ele que faz o trabalho duro que não é visto.
Sempre busco meios de melhorar o manuseio dessa ferramenta visto que ela é responsável por várias coisas em uma aplicação feita para a web, inclusive a segurança.

MongoDb

Pode se dizer que é um recipiente para que possamos guardar e recuperar dados, então se o cliente me pedir para criar um campo em que guarda registros, por exemplo, o nome e telefone de clientes ou os serviços agendados, é aqui que vou guardar essas informações.

PHP

É uma outra ferramenta que busco usar para interagir com as partes não vistas de um site, é mais antiga que o Node.js mas é muito eficaz .
Por já estar no mercado a algum tempo, existem muitos sites que usam esta tecnologia, então tenho como rotina utilizar essa tradicional linguagem dos modos mais inovadores possível.

MySql

É outro recipiente digital de dados que os sites usam, é um pouco mais tradicional e formal que o Mongodb e seu uso é um pouco diferente, mas é igualmente poderoso.
Por isso que sempre atualizo minha experiência no seu uso.
